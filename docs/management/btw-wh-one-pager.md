# BTW-WH — one-pager для руководства

## Что это

Единая цифровая платформа **склада и товарного контура**: артикулы, физическая и логическая карта склада, заявки и дефициты, конкурентный/SKU-учёт, сравнение продаж и остатков с Btrade. Клиенты: **веб** (основной) и **мобильное приложение** (Expo); данные — через **REST API**.

## Зачем бизнесу

| Боль | Как закрывает BTW-WH |
|------|----------------------|
| Данные в разных местах | Одна модель: склад ↔ номенклатура ↔ внешние источники |
| Непонятно, кто что запросил | Заявки с историей, pull, уведомления; лёгкий канал «к кассе» (kasks) |
| Слепые дефициты | Расчёт vs внешний сайт, лимиты, связь с уже созданными заявками |
| Хаос «где лежит товар» | Зоны (штрихкоды, сектора), ряды → паллеты → позиции, блоки/сегменты для админов |
| Несанкционированные действия | Роли USER / ADMIN / PRIME на маршрутах и в API |

## Сильные стороны «внутри»

Транзакции и аудит событий по заявкам; асинхронный тяжёлый расчёт дефицитов; кеши справочных данных в документах (меньше запросов, стабильный прошлый снимок); интеграция с **Telegram** (заявки, касса); маркеры версий артикулов и связь с **Btrade**.

## Куда развивать

Больше мобильных сценариев и сканирования; KPI-дашборды; углубление аналитики по срезам; автоматизация импортов и регламентных пересчётов.
